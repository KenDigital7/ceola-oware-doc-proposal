# Transcript → Proposal Pipeline

Repeatable order so you’re not reinventing the doc each time.

## Step 1 — Ingest (15–30 min)

1. Drop transcript or notes into `Context/` (or your CRM).
2. Skim once for bullets only: **urgency, who, where, story beats, budget hint, fears**.
3. Ignore small talk unless it reveals **values** (e.g. “don’t exploit my dad”).

## Step 2 — Internal brief (20–40 min)

1. Fill [01-intake-from-transcript.md](01-intake-from-transcript.md).
2. Save as `docs/project-brief-<client-slug>.md`.
3. Flag **risks**: health, legal names, licensing, unrealistic runtime vs. one day.

## Step 3 — Package decision (10 min)

1. Choose **one clear offer** OR **tiered** OR **phased milestones**—don’t mix models in one page without headers.
2. List **inclusions / exclusions** in a table (clients scan tables).
3. Park **everything else** in **Add-ons** with “quoted separately.”

## Step 4 — Draft proposal (60–90 min)

1. Copy [03-proposal-skeleton.md](03-proposal-skeleton.md).
2. Write **Executive summary** last *or* write it first then rewrite after body is done (usually tighter second pass).
3. Pull **2–4 exact phrases** from the client transcript into **Why this matters** — proves you listened.
4. Keep **Investment** page foolproof: one number, what it buys, what it doesn’t.

## Step 5 — Voice pass (15 min)

1. **Care / archive language** for family & community docs (not hype).
2. **Credibility + specs** block for institutional or high-budget clients (camera, audio, post tool).
3. Cut generic filler: if it could apply to any client, personalize or delete.

## Step 6 — Reader test (10 min)

1. Run [05-pre-send-checklist.md](05-pre-send-checklist.md).
2. Optional: paste proposal into a **fresh** chat: “What questions would a skeptical family member ask?” Patch gaps.

## Step 7 — Send kit

1. PDF or shared doc link + short email: **scope, price, one next step**.
2. Attach or link **archival checklist** and **timeline** as separate files if the proposal is long.

## Automation ideas (optional)

- **Transcription:** Otter, Descript, or phone recorder → text.
- **Templates:** Keep skeleton + pricing blocks in repo; swap tables only.
- **CRM:** Same intake fields every time (Notion / Airtable).
- **AI:** Paste intake + transcript quotes; ask for **brief** first, then **proposal** from brief only (reduces hallucination).
